# GamingOverlay

Production-oriented Windows 10/11 gaming overlay ecosystem inspired by MSI Afterburner, RTSS, Steam Overlay, Xbox Game Bar, Discord Overlay, and NVIDIA Overlay.

This repository contains:

- `Overlay.exe`: C++17 Win32 + DirectX 11 + Dear ImGui transparent overlay.
- `Launcher.exe`: C# WPF .NET 8 launcher, updater, and crash supervisor.
- `Setup.exe`: Inno Setup bootstrap installer generated by CI.
- `latest.json`: remote version manifest for GitHub Releases.
- `Overlay.zip`: release update package generated by CI.

The default runtime path is intentionally anti-cheat conservative: it renders as a transparent topmost external overlay window and collects read-only telemetry. A Present-hook extension seam exists for owned/consented processes only; the project does not ship an injector, kernel driver, memory editor, or bypass behavior.

## Architecture

```text
Setup.exe
|
+-- installs Launcher.exe, Overlay.exe, assets, config
+-- creates desktop/start menu shortcuts
+-- optionally registers HKCU auto-start
+-- launches Launcher.exe

Launcher.exe
|
+-- downloads latest.json from GitHub Releases
+-- compares semantic versions
+-- downloads Overlay.zip
+-- validates SHA-256
+-- extracts to staging
+-- patches install directory
+-- launches Overlay.exe
+-- records crash exits in logs/launcher.log

Overlay.exe
|
+-- Win32 transparent topmost window
+-- DirectX 11 renderer
+-- Dear ImGui UI layer
+-- telemetry worker thread
+-- widget registry
+-- config/theme/profile manager
+-- hotkey system
+-- crash minidumps
+-- owned-process Present-hook extension seam
```

## Overlay Features

- FPS counter with 1% low and 0.1% low calculations.
- Frametime graph using `ImGui::PlotLines`.
- CPU usage, disk throughput, RAM usage, network counters through PDH and Win32 APIs.
- GPU VRAM metadata through DXGI, with NVAPI/AMD ADL extension points.
- Draggable/resizable widgets with persisted layouts.
- Transparent always-on-top DX11 overlay window.
- Click-through mode for normal gameplay.
- `Insert`: toggle overlay.
- `F10`: settings.
- `F11`: performance mode.
- JSON config, themes, profiles, opacity, scaling, and widget positions.
- Crash log and minidump generation under `logs/`.

## Build Requirements

- Windows 10/11 x64.
- Visual Studio 2022 with Desktop development with C++.
- Windows 10/11 SDK.
- CMake 3.24+.
- .NET 8 SDK.
- Inno Setup 6 for installer builds.

## Build Overlay

From `GamingOverlay/`:

```powershell
mkdir build
cd build
cmake .. -A x64 -DOVERLAY_FETCH_DEPS=ON
cmake --build . --config Release --parallel
```

`OVERLAY_FETCH_DEPS=ON` downloads Dear ImGui and nlohmann/json. For locked-down build machines, vendor those dependencies under `overlay/third_party` and configure with `OVERLAY_FETCH_DEPS=OFF`.

## Build Launcher

```powershell
dotnet publish launcher/Launcher/Launcher.csproj -c Release -r win-x64 --self-contained true -o releases/package
```

Copy the overlay output into the package directory:

```powershell
Copy-Item build/overlay/Release/Overlay.exe releases/package/Overlay.exe -Force
Copy-Item overlay/assets releases/package/assets -Recurse -Force
Set-Content releases/package/version.txt 1.0.0
```

## Build Release Package

```powershell
powershell -ExecutionPolicy Bypass -File scripts/build-release.ps1 -Configuration Release -Version 1.0.0
```

This creates `releases/package` and `releases/Overlay.zip`.

## Build Installer

After `releases/package` exists:

```powershell
& "${env:ProgramFiles(x86)}\Inno Setup 6\ISCC.exe" setup/installer.iss
```

Output: `releases/Setup.exe`.

## Remote Update System

The launcher reads `appsettings.json`:

```json
{
  "manifestUrl": "https://github.com/USERNAME/REPO/releases/latest/download/latest.json",
  "overlayExecutable": "Overlay.exe",
  "localVersionFile": "version.txt",
  "restartOnCrash": false
}
```

`latest.json` format:

```json
{
  "version": "1.0.0",
  "download": "https://github.com/USERNAME/REPO/releases/download/v1.0.0/Overlay.zip",
  "sha256": "release-package-sha256",
  "changelog": "Release notes",
  "minLauncherVersion": "1.0.0"
}
```

Update flow:

1. Launcher starts.
2. Downloads `latest.json`.
3. Compares local `version.txt` against remote version.
4. Downloads `Overlay.zip`.
5. Validates SHA-256.
6. Extracts to a staging directory.
7. Verifies `Overlay.exe` exists.
8. Copies files into the install directory.
9. Writes the new local version.
10. Launches or relaunches the overlay.

## Release Pipeline

`.github/workflows/release.yml` runs on tags like `v1.0.0`:

1. Builds C++ overlay in Release x64.
2. Publishes WPF launcher self-contained for win-x64.
3. Stages `Overlay.exe`, `assets`, `Launcher.exe`, and `version.txt`.
4. Creates `Overlay.zip`.
5. Computes SHA-256.
6. Generates `latest.json`.
7. Builds `Setup.exe` with Inno Setup.
8. Uploads assets to GitHub Releases.

## Rendering Design

The default overlay renderer uses:

- `WS_EX_LAYERED | WS_EX_TOPMOST | WS_EX_TOOLWINDOW`.
- Optional `WS_EX_TRANSPARENT` click-through mode.
- `DwmExtendFrameIntoClientArea` for transparent composition.
- DirectX 11 double-buffered swap chain.
- Dear ImGui DX11 + Win32 backends.
- No blocking telemetry calls on the render thread.

Anti-flicker and pacing choices:

- Double buffering.
- Clear with transparent alpha each frame.
- Render state is recreated on resize.
- VSync enabled by default for stable frame pacing.
- F11 performance mode disables sync interval for lower overlay latency.
- `timeBeginPeriod(1)` improves sleep granularity.

## Telemetry Threading

Telemetry runs on one worker thread at 250 ms cadence:

- PresentMon-compatible frametime ingestion, with a local synthetic fallback for development.
- PDH counters for CPU, disk, and network.
- Win32 memory status for RAM.
- DXGI memory metadata for VRAM.
- Vendor GPU providers are intentionally isolated behind `GpuProvider`.

The render loop reads a copied snapshot, so widgets never poll PDH, WMI, NVAPI, ADL, disk, network, or HTTP directly.

## Widget Architecture

Widgets implement:

```cpp
class IWidget {
public:
    virtual ~IWidget() = default;
    virtual const char* Id() const = 0;
    virtual const char* Title() const = 0;
    virtual void Render(const Telemetry::MetricSnapshot& snapshot, Config::WidgetLayout& layout) = 0;
};
```

`WidgetRegistry` supports dynamic registration. `PluginLoader` scans `plugins/` for DLLs that export:

```cpp
extern "C" __declspec(dllexport)
bool __cdecl RegisterOverlayWidgets(WidgetRegistry* registry);
```

Production builds should add Authenticode signature verification before loading third-party plugin DLLs.

## Configuration

Runtime files:

- `assets/config.json`: profiles, hot behavior, scaling, opacity, widget positions.
- `assets/theme.json`: color theme and accents.

Layouts are updated in memory while dragging/resizing and persisted on shutdown or from the settings window.

## Vendor Telemetry

NVAPI and AMD ADL are SDK/licensing-sensitive and are not vendored here. Production integration should keep them read-only:

- GPU utilization.
- GPU temperature.
- Core/memory clocks.
- VRAM.
- Board power.

Do not expose fan, voltage, power-limit, or clock mutation paths from the overlay process unless you add explicit user consent, safety limits, logging, and vendor API error handling.

## Anti-Cheat Safety

This project avoids:

- Kernel drivers.
- Game memory editing.
- Code injection into protected games.
- Signature hiding.
- Anti-cheat bypasses.
- Process hollowing or remote thread loaders.

Preferred production modes:

- External transparent overlay for general compatibility.
- Game-owned SDK/in-process load for titles you own.
- ETW/PresentMon for frame telemetry where possible.
- Read-only vendor telemetry APIs.

Fullscreen exclusive games may block normal desktop overlays. For those cases, use a game-owned plugin/SDK path or rely on borderless fullscreen.
